# Vedic Light - 出生地点选择功能优化需求

## 需求来源

基于QQ聊天记录中的用户反馈和讨论，整理出以下优化需求。

---

## 一、问题分析

### 1.1 用户体验问题

| 问题描述 | 影响范围 | 严重程度 |
|----------|----------|----------|
| 位置输入太复杂，用户感到不耐烦 | 新用户 | 高 |
| 缺少国家-省-市层级选择功能 | 所有用户 | 高 |
| 区县级别地点无法准确识别 | 中国大陆用户 | 高 |
| 全球覆盖需要支持多城市层级 | 国际用户 | 中 |

### 1.2 当前功能痛点

1. **缺少层级选择**：用户希望通过国家→省→市的层级选择方式输入地址
2. **搜索精度不足**：用户输入区县名称时，搜索结果不优先显示区县级地点
3. **反馈不及时**：搜索等待时间较长，用户容易失去耐心
4. **地点数据不全**：中国大陆省市区县数据覆盖不足

---

## 二、优化需求

### 2.1 核心优化点

#### 需求1：区县级别地点优先显示

**需求描述**：
- 当用户输入区县名称（如"朝阳区"、"海淀区"）时，搜索结果应优先显示区县级地点
- 支持智能识别地址后缀（区、县、市、街道等）

**预期行为**：
- 用户输入"朝阳" → 优先显示"北京市朝阳区"
- 用户输入"海淀" → 优先显示"北京市海淀区"

---

#### 需求2：简化地址输入流程

**需求描述**：
- 提供更简洁的地址输入体验
- 支持直接输入完整地址（如"中国 北京市 朝阳区"）
- 自动解析地址层级

**预期行为**：
- 支持"城市 区县"格式输入
- 支持"区县"单独输入（自动匹配上级城市）

---

#### 需求3：优化搜索结果排序

**需求描述**：
- 按照地址精度排序：区县 > 市 > 省 > 国家
- 相同精度下，按照匹配度排序

**排序优先级**：
1. 区县/街道级别（最高）
2. 城市级别
3. 省/州级别
4. 国家级别（最低）

---

#### 需求4：增强中国大陆地址支持

**需求描述**：
- 补充中国大陆主要城市的区县数据
- 支持常用区县名称的别名映射

**覆盖范围**：
- 直辖市：北京、上海、天津、重庆
- 省会城市：广州、深圳、杭州、成都等
- 主要地级市的区县数据

---

#### 需求5：智能地址补全

**需求描述**：
- 当用户输入不完整地址时，自动补全可能的地址
- 例如：输入"北京" → 显示北京市及下属区县

---

### 2.2 用户体验优化

#### 需求6：快速选择常用地点

**需求描述**：
- 在搜索框下方显示热门城市/地区快捷选择
- 支持按地区分类（如华北、华东、华南等）

**预期行为**：
- 点击热门城市标签可快速填充位置
- 热门城市列表可配置

---

#### 需求7：实时搜索反馈

**需求描述**：
- 搜索过程中显示加载动画
- 搜索结果为空时显示友好提示
- 提供重新搜索或手动输入坐标的选项

---

## 三、技术实现建议

### 3.1 后端优化

1. **增强本地地点数据**：
   - 扩展 `reference_locations.json`，添加中国大陆区县数据
   - 支持别名映射（如"帝都" → "北京"）

2. **优化搜索算法**：
   - 增加精度评分机制
   - 优先返回高精度匹配结果

3. **缓存热门搜索**：
   - 缓存高频搜索词结果
   - 提高响应速度

### 3.2 前端优化

1. **实现层级选择组件**：
   - 创建 `LocationSelector` 组件，包含三个联动下拉框
   - 实现搜索模式和层级选择模式的切换逻辑
   - 选择完成后自动填充位置信息（名称、坐标、时区）

2. **状态管理**：
   - 维护 `selectedCountry`, `selectedState`, `selectedCity` 状态
   - 实现级联清空逻辑（选择国家时清空省和市）

3. **智能输入提示**：
   - 实时显示搜索进度
   - 提供输入建议

4. **结果展示优化**：
   - 按精度分组显示结果
   - 高亮匹配关键词

5. **快捷选择面板**：
   - 热门城市快速选择
   - 历史记录快速访问

## 五、验收标准

### 5.1 功能验收

| 测试场景 | 预期结果 |
|----------|----------|
| 输入"朝阳区" | 优先显示北京市朝阳区 |
| 输入"海淀" | 优先显示北京市海淀区 |
| 输入"中国 上海 浦东" | 正确匹配上海市浦东新区 |
| 输入"广州天河" | 优先显示广州市天河区 |
| 搜索结果为空 | 显示友好提示并提供手动输入选项 |

### 5.2 性能验收

- 搜索响应时间 < 500ms
- 搜索结果显示时间 < 300ms
- 支持300ms防抖搜索
